# 液态玻璃效果(Liquid Glass)制作教程

> 本教程整理自 Outpace Studios 的文章《Liquid glass for the web》(https://glass.outpacestudios.com/)。
> 核心论点:**玻璃的本质是折射(光线弯折),而不是模糊**。市面上绝大多数"液态玻璃"实际只是"带亮边的磨砂板"——一个 `backdrop-filter: blur()` 加一个亮色边框。去掉那层亮边,它就只是一块磨砂面板,不是玻璃。

---

## 0. 一句话总览

> 真正的玻璃 = **用光学公式算出每个像素的折射位移** + **把背景渲染一份拷贝送进 SVG 位移滤镜**。前者决定"弯多少",后者保证"在所有浏览器里都能弯"。

---

## 1. 为什么大多数"液态玻璃"是假的

玻璃会让穿过它的光线弯折,弯折最剧烈的地方在边缘——因为那里表面曲率最大。这个"弯折"才是玻璃的标志。

- **拿掉弯折** → 你得到一块磨砂面板(frosted panel)。
- **市面常见的做法**:`backdrop-filter: blur()` + 一个亮色边框。这只是"看起来像玻璃的照片",不是玻璃。
- **本教程的做法**:从光学物理出发计算折射,而不是用渐变去"画"一个接近的样子。

---

## 2. 光学原理:弯折是算出来的,不是画出来的

整个效果归结为**每个像素一个值**:这个像素处的背景应该被横向推移多远。

- 用软辐射渐变画这个值 → 足够"玻璃感",大多数 demo 停在这一步。
- 从"表面形状"计算这个值 → 得到真实透镜会产生的扭曲。

### 2.1 透镜模型:凸形 squircle 圆顶

把透镜建模成一个物理对象:

- 一个 **squircle(超椭圆)圆顶**,中部平坦,边缘通过一条窄"边缘带"快速向下弯曲。
- squircle 就是 Apple 用来做圆角的那种超椭圆曲线。
- 中心平坦 → 斜率为 0 → 不弯折;边缘陡峭 → 斜率大 → 弯折集中在边缘。
- 这正是**厚玻璃**的真实行为:中心清晰,边缘弯折。

### 2.2 用斯涅尔定律(Snell's Law)算折射

对每个像素:

1. 取圆顶在该处的**斜率**(slope)。
2. 把一条垂直向下的视线,按 **折射率 1.5(真实玻璃)** 用斯涅尔定律折射穿过表面。
3. 折射后光线的**横向偏移量**就是位移(displacement)。
4. 位移方向沿"圆角矩形 SDF(带符号距离场)的法向量"。
5. 用蓝色通道承载**镜面高度**,用来生成边缘高光。

核心代码:

```js
// 表面是 squircle 圆顶。
// x = 从边缘向内的深度(0 = 边缘, 1 = 平坦中心)
// slope = 表面倾角
const slope = (1 - x) ** 3 / (1 - (1 - x) ** 4) ** 0.75
const thetaI = Math.atan(slope)                    // 入射角
const thetaT = Math.asin(Math.sin(thetaI) / 1.5)   // 折射角(斯涅尔定律)
const bend = Math.sin(thetaI - thetaT)             // 中心为 0,边缘最大

// 把 bend 沿圆角矩形 SDF 法向量 (nx, ny) 投射,
// 然后写入位移图的 R/G 通道(128 = 原地不动)
map.r = 128 + nx * bend * gain
map.g = 128 + ny * bend * gain
```

### 2.3 位移图驱动 SVG 滤镜

这张位移图(map)驱动**一个** SVG 滤镜原语:`feDisplacementMap`。

- 对每个像素,它查表取出位移量,把该处内容移动相应距离。
- 位移值直接来自上面的折射数学 → 输出就是真实玻璃会产生的弯折,而不是"调到看起来差不多"的渐变。

```html
<svg width="0" height="0" style="position:absolute">
  <filter id="liquid-glass" color-interpolation-filters="sRGB">
    <feImage href="blob:..." result="map" />
    <feDisplacementMap in="SourceGraphic" in2="map" scale="80"
                       xChannelSelector="R" yChannelSelector="G" />
  </filter>
</svg>
```

## 3. 跨浏览器的关键:折射"拷贝",而不是"背景"

光学那一半是简单的。难的那一半是一个纯 Web 问题:**怎么把真实的背景像素送进滤镜,而且在所有浏览器里都能跑**。

### 3.1 显而易见的路线是个陷阱

`backdrop-filter` 可以跑 SVG 位移滤镜——**但只在 Chromium 里**。

- Safari、Firefox 接受 `backdrop-filter` 这个属性,但会**静默丢弃 SVG 部分**,只剩一个扁平模糊。
- 这一个缺口,就是为什么大多数 Web "液态玻璃"在 Chrome 里是玻璃、在其他浏览器里是磨砂。

### 3.2 解法:不滤背景,滤"拷贝"

`feDisplacementMap` 通过普通的 CSS `filter` 属性在**所有浏览器**都能跑;只有 `backdrop-filter` 是 Chromium 专属。

所以做法是:

1. 把背景**再渲染一份**(拷贝)。
2. 把这份拷贝**1:1 反向定位**到透镜正下方。
3. 对拷贝施加位移滤镜(弯曲它)。
4. 透镜下方的真实界面**从不被滤镜处理**,保持完全可交互;弯曲的拷贝只是作为"一层光"覆盖其上。

> 这一手"滤拷贝而非滤背景"来自 Aave 的文章《Building Glass for the Web》(https://aave.com/design/building-glass-for-the-web)。
> 光学计算是 Outpace 自己的;这个架构让光学能在任何地方运行,无需 WebGL 或截图。

```jsx
<GlassScene content={<Backdrop />}>
  <nav>{triggers}</nav>
  {open && (
    <GlassLens x={x} y={y} width={W} height={h}>
      {menuItems}
    </GlassLens>
  )}
</GlassScene>
```

- `GlassScene`:渲染背景一次并共享它。
- `GlassLens`:把一份反向定位的拷贝投进透镜盒子并弯曲它。
- 拷贝逐帧追踪背景 → 玻璃实时折射**动态移动**的内容。滚动或让背景图循环,透镜会持续弯折它后面的东西。

### 3.3 为什么是"拷贝"而不是其他更炫的方案

| 方案 | 问题 |
|------|------|
| WebGL 截图 | 光学真实,但把页面冻结成一张过时、不可选中的纹理 |
| Firefox `element()` | 能把活动元素渲染成图,正是所需——但**只有 Firefox 有** |
| Houdini Paint Worklet | **禁止读取已有像素** |
| 纯 CSS | 根本碰不到背景,只是"玻璃的照片" |
| **活动拷贝 + 真实滤镜** | **唯一既诚实又通用的路线** |

## 4. 那些会咬人的细节(跨浏览器落地)

把效果跨浏览器跑起来,是一串小而不明显的坑(部分来自 Outpace,部分 Aave 已指出):

### 4.1 位移图必须用 `blob:` URL,不能用 `data:` URI

- 每篇教程都告诉你把位移图内联成 `data:` URI。
- **真实 WebKit 会静默拒绝**在 `feImage` 里加载 `data:` URI → 图永远到不了 → 玻璃塌缩成扁平磨砂。
- **`blob:` URL 在所有浏览器都能加载。**

### 4.2 强制 sRGB,不要用默认的 linearRGB

- 滤镜默认跑在 `linearRGB` 色彩空间,会悄悄改变"某个灰度值位移多远"。
- 在 `<filter>` 上设 `color-interpolation-filters="sRGB"`,让位移图"说的就是它说的"。

### 4.3 Safari 的两个让步(Aave 指出)

1. **按 `id` 缓存滤镜输出**:一个会动画的透镜,每次重建都需要一个**新 id**,否则它会冻结在第一帧。
2. **滤镜源图尺寸上限**:Safari 限制单个滤镜能处理的源图大小。折射拷贝在光栅化前**必须先裁剪到透镜盒子**。给它一个全视口场景,滤镜会静默地产出空。

### 4.4 不同浏览器跑不同的滤镜链

- **Chromium**:SVG 滤镜跑在 GPU 上 → 完整效果(三次不同缩放的位移 pass 产生边缘色散,加上镜面 pass)**几乎免费**。
- **Safari**:SVG 滤镜跑在软件层;这里背景在不停平移 → 滤镜**每一帧都要重新光栅化**;第二个用于色散的 pass 超出预算会掉帧。
  → Safari 只跑**一次 pass**:同样的圆顶折射 + 同样的边缘高光,去掉那层淡淡的色彩分裂。
- **永远不会回退成扁平模糊**——只是按各自能负担的尺度呈现同一种材质。

## 5. 交互:一片活着的透镜

- **只有一片透镜**,不是每个菜单一片。
- 在导航上移动时,透镜**滑向下一个菜单项**并**改变形状**到那个下拉的尺寸——一片玻璃在移动中改变大小和位置。
- 运动是**弹簧驱动且可打断**的:快速扫过触发器,它会落到你最后停下的那一项。
- 菜单行在透镜到达后**快速错峰升起**。
- 你读到的是**连续性**:玻璃作为一个移动的整体被持有。

---

## 6. 默认无障碍

折射是"运动 + 半透明",不是所有人都想要这两者。

- **`prefers-reduced-motion`**:透镜停止动画,直接切到新位置(而不是弹簧过去)。
- **`prefers-reduced-transparency`**(系统级"我在玻璃上看不清文字"开关):效果**完全去掉折射**,换成不透明面板,文字坐在纯色表面上满对比度。
- **玻璃是增强项**;它下面的界面没有它也能正常工作。

```css
@media (prefers-reduced-motion: reduce) {
  .glass-lens { transition: none; }
}
@media (prefers-reduced-transparency: reduce) {
  .glass-lens { filter: none; background: var(--panel-solid); }
}
```

## 7. 引擎总览

整个引擎 = **两个组件 + 一个滤镜原语**:

| 部分 | 职责 |
|------|------|
| `GlassScene` | 共享背景 |
| `GlassLens` | 通过上面的光学把背景拷贝弯曲 |
| `feDisplacementMap` | 唯一的滤镜原语,执行位移 |

> 折射从来不是难的那部分。**把诚实的背景像素送进一个真实滤镜,且在所有浏览器里都能跑**,才是难的。这件事解决一次,玻璃就只有几百行代码。
> 你得到的不是一个"在演玻璃的模糊",而是**真实折射**,并且在 Safari 和 Firefox 里也能跑,不只是 Chrome。

——Outpace Studios

## 8. 快速版:纯 CSS"伪液态玻璃"(用于预算有限的场景)

如果你的项目无法承受 SVG 位移滤镜 + 背景拷贝的工程量,可以用下面这个**纯 CSS 的"玻璃感"**作为降级。它不是真折射,但视觉上接近,且零 JS、跨浏览器无差异:

```css
.liquid-glass {
  background: rgba(255, 255, 255, 0.01);
  background-blend-mode: luminosity;
  backdrop-filter: blur(4px);
  -webkit-backdrop-filter: blur(4px);
  border: none;
  box-shadow: inset 0 1px 1px rgba(255, 255, 255, 0.1);
  position: relative;
  overflow: hidden;
}

/* 用 mask 抠出一条"边缘渐变亮边",模拟玻璃边缘高光 */
.liquid-glass::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  padding: 1.4px;
  background: linear-gradient(
    180deg,
    rgba(255, 255, 255, 0.45) 0%,
    rgba(255, 255, 255, 0.15) 20%,
    rgba(255, 255, 255, 0) 40%,
    rgba(255, 255, 255, 0) 60%,
    rgba(255, 255, 255, 0.15) 80%,
    rgba(255, 255, 255, 0.45) 100%
  );
  -webkit-mask: linear-gradient(#fff 0 0) content-box, linear-gradient(#fff 0 0);
  -webkit-mask-composite: xor;
  mask-composite: exclude;
  pointer-events: none;
}
```

**适用场景**:导航栏 CTA 按钮、卡片、小尺寸面板。
**不适用场景**:需要真实折射动态背景的大型透镜(如下拉菜单覆盖在循环视频上)——这种请用第 2~4 节的完整方案。

## 9. 实现路线图(从零到上线)

1. **决定透镜用在哪**:导航下拉?卡片悬停?整页遮罩?(范围越小越容易落地)
2. **写位移图生成器**:用 Canvas 2D 按第 2.2 节公式生成 R/G/B 通道,导出为 `Blob` → `URL.createObjectURL()` 得到 `blob:` URL。
3. **搭 SVG 滤镜**:`feImage` 引用 `blob:` URL,接 `feDisplacementMap`,设 `color-interpolation-filters="sRGB"`。
4. **实现背景拷贝机制**:`GlassScene` 渲染背景一次,`GlassLens` 反向定位一份拷贝,把滤镜作用在拷贝上。
5. **Safari 适配**:透镜每次重建换新 `id`;拷贝裁剪到透镜盒子;Safari 只跑单 pass。
6. **运动**:用弹簧动画驱动透镜的位置/尺寸;菜单行错峰升起。
7. **无障碍**:加 `prefers-reduced-motion` 与 `prefers-reduced-transparency` 媒体查询。
8. **测试矩阵**:Chrome(完整效果含色散)、Safari(单 pass 无色散)、Firefox(同 Safari)。
9. **降级**:在不支持 SVG 滤镜的环境,回退到第 8 节的纯 CSS 版本。
